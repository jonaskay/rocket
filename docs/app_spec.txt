<project_specification>
  <project_name>Rocket – Training Platform</project_name>

  <overview>
    A multi-tenant training platform that allows organizations to manage
    trainers and training content. Super admins oversee client accounts. Client
    account admins manage their organization and trainers. Trainers build master
    trainings (with uploaded slides, prerequisite assets, and rich-text HTML
    exercises), then spin off versioned training sessions from them. Each
    session is an immutable snapshot of the master training at the moment of
    generation and can be password-protected for controlled access.
  </overview>

  <technology_stack>
    <framework>Ruby on Rails (full-stack, not API mode)</framework>
    <ruby_version>3.4.5</ruby_version>
    <rails_version>8.1.2</rails_version>

    <i18n>
      Rails built-in i18n with default_locale :en and available_locales [:en].
      All user-facing strings extracted to namespace-scoped YAML locale files
      under config/locales/. Views use lazy lookup (t(".key")); controllers use
      fully qualified keys.
    </i18n>

    <database>PostgreSQL</database>

    <frontend>
      <views>ERB templates with Rails layouts</views>
      <styling>Tailwind CSS (using tailwindcss-rails gem)</styling>

      <javascript>
        Hotwire: Turbo Drive + Turbo Frames + Turbo Streams for page updates
        without full reloads; Stimulus for JavaScript behavior
      </javascript>

      <rich_text_editor>
        Trix via Action Text for exercise HTML authoring (built into Rails)
      </rich_text_editor>

      <file_uploads>
        Direct-to-cloud uploads via Active Storage's direct upload feature
        (generates signed GCS upload URLs)
      </file_uploads>
    </frontend>

    <authentication>
      Rails authentication generator (rails generate authentication)
    </authentication>

    <file_storage>
      Active Storage configured with Google Cloud Storage (GCS) service. Direct
      uploads use GCS signed URLs generated by Active Storage. Secure downloads
      use short-lived signed GCS URLs (15-minute expiry).
    </file_storage>

    <email>
      Action Mailer configured to deliver via Mailgun (using the mailgun-ruby
      gem or Mailgun's SMTP relay). Used for trainer invitation emails and
      password reset emails.
    </email>

    <background_jobs>Solid Queue</background_jobs>
    <port>Only launch on port {port}</port>
  </technology_stack>

  <roles>
    <role name="super_admin">
      Global administrator. Identified by super_admin: true on the User model.
      Not tied to any client account. Can create and delete client accounts. Can
      create the first admin user for a client account.
    </role>

    <role name="account_admin">
      Belongs to exactly one client account. Can update their organization's
      name. Can invite, activate, and remove trainers within their account.
      Cannot access other client accounts.
    </role>

    <role name="trainer">
      Belongs to exactly one client account. Can create, edit, and delete master
      trainings within their account. Can generate training sessions from master
      trainings. Can manage (rename, set/change password, delete) their own
      training sessions.
    </role>
  </roles>

  <core_features>
    <super_admin_panel>
      - Dedicated /admin area accessible only to users with super_admin: true -
      Admin home page at /admin (Admin::HomeController#index) - List all client
      accounts with name, creation date, and trainer count - Create new client
      account: name + initial account_admin email/password - Delete client
      account (cascades: removes all users, trainings, sessions, assets) - View
      trainers and account admins for any client account (read-only)
    </super_admin_panel>

    <account_management>
      - Account settings page for account_admins - Edit organization name -
      Trainer roster: list all trainers with email and status
      (active/inactive/pending_password_change); name will be shown once
      first_name/last_name columns are added - Invite new trainer: enter name +
      email; Rails generates a secure random password, creates the user with
      status pending_password_change, and sends an invitation email via Mailgun
      containing a one-time setup link - Deactivate / reactivate trainer (soft
      deactivation; deactivating destroys all active sessions immediately;
      deactivated trainers cannot log in and are rejected at session resume;
      pending_password_change trainers cannot be deactivated) - Remove trainer
      permanently (hard delete with confirmation modal)
    </account_management>

    <authentication>
      - Login page for all roles (email_address + password), scaffolded by rails
      generate authentication - Cookie-based sessions stored server-side in the
      sessions table - After sign-in, redirect super_admin users to
      admin_root_path (/admin) - Password reset flow via emailed token (Action
      Mailer + Mailgun) - Invited trainers set their password via the invite
      link (/passwords/:token/edit) before their first login;
      PasswordsController#update sets status to active upon success - Forced
      password change on direct login: if a trainer with status
      pending_password_change logs in directly (without using the invite link),
      they are redirected to /account/password/edit and cannot navigate to any
      other page until they set a new password; upon successful update their
      status changes to active and they are redirected to the home page - The
      Authentication concern enforces a require_password_change_if_pending
      before_action (placed after require_authentication) that redirects
      pending_password_change users to /account/password/edit; exceptions:
      Account::PasswordsController and SessionsController -
      Account::PasswordsController inherits from ApplicationController (not
      Account::ApplicationController) and handles the forced password change
      form; guards with require_password_change before_action that redirects
      active users away from the change-password page - Inactive users (status:
      inactive) are blocked from logging in and see a deactivation message -
      Logout destroys the session record and clears the cookie -
      Admin::ApplicationController inherits from ApplicationController and adds
      require_admin before-action (checks super_admin?) - Non-super_admin
      authenticated users redirected away from /admin with flash notice
    </authentication>

    <master_trainings>
      - Trainers see a dashboard listing all master trainings for their account
      - Create master training: title and optional description - Edit master
      training metadata (title, description) at any time - Delete master
      training (only if no sessions have been generated from it; otherwise show
      warning)
      <slides>
        - Upload one or more slide files (PDF or PPTX) per master training -
        Files uploaded via Active Storage direct upload to Google Cloud Storage
        - Each file has: display name, upload date, file size, and a download
        button (generates a short-lived signed GCS URL served through Rails) -
        Reorder slides via drag-and-drop (Stimulus + Stimulus Sortable; persists
        position via a PATCH request with Turbo Stream response) - Delete
        individual slide files
      </slides>

      <prerequisite_assets>
        - Upload prerequisite files (any file type: PDF, video, ZIP, etc.) -
        Same Active Storage direct upload flow as slides - Each asset has:
        display name, description (optional), upload date, file size, and a
        download button (signed GCS URL) - Reorder assets via drag-and-drop
        (same Stimulus + Stimulus Sortable pattern) - Delete individual assets
      </prerequisite_assets>

      <exercises>
        - Add one or more exercises per master training - Each exercise has:
        title and HTML body authored with Action Text (Trix editor) - Trix
        supports: headings, bold, italic, bullet lists, numbered lists, links,
        image attachments (uploaded to GCS via Active Storage), and code
        formatting - Exercise content is stored via Action Text's rich_text
        association (sanitized automatically) - Reorder exercises via
        drag-and-drop (Stimulus + Stimulus Sortable) - Preview exercise as it
        will appear to participants (rendered in a Turbo Frame panel) - Delete
        exercises
      </exercises>

      <versioning>
        - Every time a training session is generated, the current state of the
        master training is captured as an immutable version snapshot. - A
        version records: title, description, ordered list of slide file
        references, ordered list of prerequisite asset references, and full
        exercise HTML content. - Trainers can view a read-only history of all
        versions with timestamp and session count. - Editing the master training
        does NOT alter existing sessions; changes only affect the next session
        generated.
      </versioning>
    </master_trainings>

    <training_sessions>
      - From the master training detail page, trainers click "Generate Session"
      - On generation: 1. A version snapshot of the master training is created
      (see versioning above) 2. A training session record is created linked to
      that version snapshot 3. Session gets an auto-generated name: "{Master
      Training Title} – Session {N}" (editable) - Session list page shows all
      sessions for a master training with: name, generated date, version number,
      password status, and a share link - Rename a session - Set or change
      session password (bcrypt-hashed; stored on the session record) - Remove
      session password (make session open-access) - Delete session (confirmation
      required; does NOT delete the master training or other sessions)
      <session_view>
        - A public-facing session view accessible via a unique URL slug - If
        password-protected, show a password entry gate before revealing content
        - Session view renders the snapshot content: - Slides listed with
        download buttons (served via expiring presigned URLs) - Prerequisite
        assets listed with download buttons - Exercises rendered as styled HTML
        pages, navigable by tab or sidebar - Session view is read-only; no
        editing possible from this view
      </session_view>
    </training_sessions>
  </core_features>

  <database_schema>
    <tables>
      <clients>
        - id (bigint, primary key) - name (string, not null) - created_at,
        updated_at
      </clients>

      <users>
        - id (bigint, primary key) - client_id (bigint, fk → clients, nullable
        for super_admin) - email_address (string, unique, not null) -
        password_digest (string, not null) - first_name, last_name (string) -
        super_admin (boolean, not null, default: false) - client_admin (boolean,
        not null, default: false) - status (enum: active | inactive |
        pending_password_change) - created_at, updated_at
      </users>

      <sessions>
        (Generated by rails generate authentication; token stored in signed
        cookie)
      </sessions>

      <master_trainings>
        - id (uuid, primary key) - client_id (uuid, fk → clients) - trainer_id
        (uuid, fk → users) - title (string, not null) - description (text) -
        created_at, updated_at
      </master_trainings>

      <master_training_versions>
        - id (uuid, primary key) - master_training_id (uuid, fk →
        master_trainings) - version_number (integer, auto-incremented per
        master_training) - title (string) - description (text) - created_at
        (Immutable once created; asset and exercise snapshots stored in child
        tables)
      </master_training_versions>

      <slides>
        - id (uuid, primary key) - master_training_id (uuid, fk →
        master_trainings) - display_name (string) - cloud_key (string, not null)
        — object key in cloud storage - file_size_bytes (bigint) - content_type
        (string) - position (integer) - created_at, updated_at
      </slides>

      <slide_snapshots>
        - id (uuid, primary key) - master_training_version_id (uuid, fk →
        master_training_versions) - slide_id (uuid, fk → slides, nullable —
        original may be deleted later) - display_name (string) - cloud_key
        (string) — same cloud key as original; file is not re-copied -
        file_size_bytes (bigint) - content_type (string) - position (integer)
      </slide_snapshots>

      <prerequisite_assets>
        - id (uuid, primary key) - master_training_id (uuid, fk →
        master_trainings) - display_name (string) - description (text) -
        cloud_key (string, not null) - file_size_bytes (bigint) - content_type
        (string) - position (integer) - created_at, updated_at
      </prerequisite_assets>

      <prerequisite_asset_snapshots>
        - id (uuid, primary key) - master_training_version_id (uuid, fk →
        master_training_versions) - prerequisite_asset_id (uuid, fk →
        prerequisite_assets, nullable) - display_name (string) - description
        (text) - cloud_key (string) - file_size_bytes (bigint) - content_type
        (string) - position (integer)
      </prerequisite_asset_snapshots>

      <exercises>
        - id (uuid, primary key) - master_training_id (uuid, fk →
        master_trainings) - title (string, not null) - position (integer) -
        created_at, updated_at (Rich text body stored via Action Text:
        action_text_rich_texts table with record_type "Exercise" and name
        "body"; image attachments via Active Storage)
      </exercises>

      <exercise_snapshots>
        - id (uuid, primary key) - master_training_version_id (uuid, fk →
        master_training_versions) - exercise_id (uuid, fk → exercises, nullable)
        - title (string) - body_html (text) — rendered HTML copied from Action
        Text body at snapshot time - position (integer)
      </exercise_snapshots>

      <training_sessions>
        - id (uuid, primary key) - master_training_id (uuid, fk →
        master_trainings) - master_training_version_id (uuid, fk →
        master_training_versions) - trainer_id (uuid, fk → users) - name
        (string, not null) - slug (string, unique, not null) — URL-safe
        identifier for public access - password_digest (string, nullable) -
        created_at, updated_at
      </training_sessions>
    </tables>
  </database_schema>

  <key_workflows>
    <workflow name="Generate Training Session">
      1. Trainer clicks "Generate Session" on a master training (Turbo confirm
      dialog) 2. POST to /master_trainings/:id/training_sessions 3. Rails
      service object atomically: a. Increments version_number and creates
      master_training_versions record b. Copies all current slide records →
      slide_snapshots (same GCS object key, no file re-copy) c. Copies all
      current prerequisite_asset records → prerequisite_asset_snapshots d.
      Renders each exercise's Action Text body to HTML → exercise_snapshots e.
      Creates training_session linked to the new version; assigns URL-safe slug
      4. Redirects to new session's edit page with a success flash notice 5.
      Trainer can optionally rename and/or set a password immediately
    </workflow>

    <workflow name="Access a Training Session (Public)">
      1. Participant visits /s/:slug 2. If session has no password: full
      snapshot content is rendered immediately 3. If session is
      password-protected: redirect to /s/:slug/unlock (password gate form) 4.
      Participant submits password → Rails verifies with bcrypt → sets a signed,
      httpOnly cookie scoped to /s/:slug (e.g. session_access_slug) with 24-hour
      expiry 5. Redirect back to /s/:slug; before_action checks cookie on every
      request to this route 6. Content rendered: slides and assets as download
      buttons (each click hits /training_sessions/:id/download/:attachment_id
      which redirects to a fresh signed GCS URL), exercises as sanitized HTML
      panels navigable by sidebar
    </workflow>

    <workflow name="Invite Trainer">
      1. Account admin fills in trainer first name, last name, and email 2.
      Rails creates user with role: trainer, status: pending_password_change,
      and a SecureRandom password_digest placeholder 3. Rails generates a
      signed, time-limited invite token (valid 72 hours) 4.
      TrainerInvitationMailer delivers invite email via Mailgun with a link to
      /passwords/:token/edit (password reset flow) 5. Trainer clicks link, sets
      their password; status updated to active
    </workflow>
  </key_workflows>
