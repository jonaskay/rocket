Based on `docs/app_spec.txt`, create a file called `feature_list.json` with detailed end-to-end test cases. This file is the single source of truth for what needs to be built. Format is available in `docs/feature_list.example.json`.

Requirements for feature_list.json:
- Both "functional" and "style" categories
- Mix of narrow tests (2-5 steps) and comprehensive tests (10+ steps)
- Order features by priority (fundamental features first)
- Evaluate value of "passes" against system and integration tests
- Cover every feature in the spec exhaustively
